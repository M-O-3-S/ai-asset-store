# 임베디드 C 리팩토링 스킬

## 목적
SOLID 원칙을 기반으로 임베디드 C 코드를 체계적으로 리팩토링합니다.

## 적용 규칙

### 1. HAL(Hardware Abstraction Layer) 분리
- 하드웨어 직접 접근 코드를 HAL 인터페이스로 분리
- `hal_gpio.h`, `hal_uart.h` 등 명확한 인터페이스 헤더 정의
- 애플리케이션 레이어는 HAL을 통해서만 하드웨어 접근

### 2. 전역 변수 최소화
- 전역 변수는 모듈 내부 `static`으로 제한
- 필요한 경우 getter/setter 함수 제공
- 공유 상태는 구조체로 캡슐화

### 3. 매직 넘버 제거
- 모든 상수는 `#define` 또는 `enum`으로 명명
- 레지스터 비트 마스크는 의미 있는 이름 부여

### 4. 단위 테스트 가능 구조
- 비즈니스 로직과 하드웨어 의존성 분리
- 함수는 50줄 이내로 제한
- 순환 복잡도 10 이하 유지

## 사용 방법
이 스킬을 Cline/Cursor에 적용하면 위 규칙에 따라 C 파일을 자동으로 분석하고 리팩토링 제안을 생성합니다.
